## Introduction

This "Web3 Frontend Bootcamp" is jointly initiated by W3F and OpenBuild, co-hosted by Artela and Morph, and supported by theGraph, Vara, and Chainlink ecosystems. Through systematic courses and a collaborative learning mechanism, it helps more frontend developers smoothly enter the Web3 field. At the same time, this bootcamp is practice-oriented, guiding you step-by-step to familiarize yourself with and master Web3 frontend code development.

## How to Participate

**⚠️⚠️⚠️ Please read this section completely before starting!**

1. Register an OpenBuild account (https://openbuild.xyz);

2. Contact **fxxkol** to join the WeChat group;

3. Submit a PR according to "[Event Registration](#Event Registration)" to complete the registration.

### Task Submission

In addition to the text instructions below, you can also watch the video "[OpenBuild Web3 Frontend Bootcamp Assignment Submission Demonstration](https://www.bilibili.com/video/BV11gVxeuE7Y)".

#### Preliminary Steps

First, click "Fork" in the top right corner of the page to create a backup in your own repository. Then, clone the backup repository to your local machine using `git clone`. All subsequent operations should be performed within your own repository.

If you are unfamiliar with Git, you can download [GitHub Desktop](https://desktop.github.com/) and use the GUI.

#### Event Registration

Copy the `github_id` folder from the `members` folder. Rename the new folder to your GitHub ID, fill in the information in the `readme.md` file, submit your PR, and complete the registration.

**Do not delete the `github_id` folder; only submit the newly created folder!**

#### Homework

Each lesson includes a corresponding practical task. You need to create task folders under the `members/[github_id]` folder using the naming convention `task[n]`, such as `task1`, `task2`, etc.

All task code must be fully open source and publicly available. Each task requires a separate pull request (PR).

### Submission Guidelines

After completing each task, submit a pull request after your own repository's `commit`. You may only modify your own files; you cannot modify other people's files.

**⚠️⚠️⚠️** PRs that do not conform to the guidelines will be labeled with "[invalid](https://github.com/openbuildxyz/Web3-Frontend-Bootcamp/pulls?q=is%3Aopen+is%3Apr+label%3Ainvalid)". Please submit your PRs according to the requirements!

#### PR Naming

Registered PRs should be named `init: user-name`, for example, `init: Beavnvvv`.

Task PRs should be named `task[n]: user-name`, for example: `task0: Beavnvvv`.

For tasks that require writing web pages, please upload a screenshot of the result in the description.

**Please do not merge registration and task submission content together, nor submit multiple tasks together!**

**#### Enabling Multiple PRs**

Many people are highly enthusiastic about learning and eager to complete all available tasks; however, due to insufficient review staff, PR merging is relatively slow. Submitting PRs in a "single-threaded" manner will affect the progress of learners.

In fact, it is possible to enable multiple PRs to complete tasks in a "multi-threaded" manner—

After forking this repository to your own space, create corresponding feature branches based on the `main` branch, such as: `init`, `task1`.

After completing the corresponding task on a feature branch, submit a PR; switch to other feature branches to continue working on the task, then submit another PR, and so on.
